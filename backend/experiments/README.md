# HITL experiment harness

Implementation of the post-hoc-rating vs self-escalation comparison
described in `/Users/celin/.claude/plans/mossy-twirling-rabin.md`. This
directory lives on `experiment/hitl-base` and is inherited unchanged by:

- `experiment/hitl-postreview` — Approach A (post-hoc rating)
- `experiment/hitl-escalate`   — Approach B (mid-run self-escalation)

The harness drives the backend over HTTP, so the same Python entry point
runs on all three branches — the branch's submodule pin + backend code
decides which endpoints exist.

## Layout

| File | Purpose |
|---|---|
| `seed_fixture.py`  | Insert a synthetic collection with 6 known defect categories + 2 near-miss fields. Pure `generate_docs` + DB-write `seed_collection`. Emits `ground_truth.json`. |
| `ground_truth.json`| Generated by `seed_fixture.py`. Read by the oracle and the summariser. |
| `oracle.py`        | Rule-based rater with a category synonym map (the agent calls null_rate / missing_field_rate / null_or_missing_rate for the same thing). |
| `harness.py`       | Per-arm orchestrator. Triggers `/argus/run`, polls, oracle-labels findings, posts arm-specific verdicts, writes per-run JSON. |
| `metrics.py`       | Pure: precision / recall / F1 / Brier / reliability buckets from snapshots. |
| `summarize.py`     | Walks `results/`, emits `summary.md` + `summary.json` with per-arm tables, F1 trajectory, and Arm B's reliability curve. |
| `tests/`           | 36 unit tests covering generator determinism, oracle synonyms, harness flow per arm (via `httpx.MockTransport`), metrics math, summariser output. |

## End-to-end

### Local mode (recommended for the experiment proper)

The backend recognises a `LOCAL_MONGO_URI` env var that bypasses Azure
OBO + ARM lookups and uses the named MongoDB instance instead. The
harness and seed script auto-detect it and default `--cosmos-account` to
the sentinel `"local"`, `--bearer-token` to `"local-mode"`, and
`--connection-string` to `$LOCAL_MONGO_URI`.

```bash
# Start a local MongoDB (Docker, brew, whatever — only Mongo wire protocol matters).
docker run -d --name hitl-mongo -p 27017:27017 mongo:7

# Start QueryPal's backend with the bypass on. Postgres still required for
# ReportStore persistence; the existing DB_* env vars are untouched.
export LOCAL_MONGO_URI='mongodb://localhost:27017'
uvicorn main:app --reload   # from backend/

# Seed the fixture. The defaults come from LOCAL_MONGO_URI.
python -m backend.experiments.seed_fixture \
  --database hitl_test --collection hitl_eval \
  --docs 15000 --seed 42
```

### Production / Cosmos mode

```bash
# 0. Seed the fixture once. Use a dedicated Cosmos test collection — the
#    script DROPS the target before inserting.
python -m backend.experiments.seed_fixture \
  --connection-string 'mongodb://...' \
  --cosmos-account '/subscriptions/.../databaseAccounts/<acct>' \
  --database hitl_test --collection hitl_eval \
  --docs 15000 --seed 42

# 1. For each arm, checkout its branch + update the submodule, then run.
#    The bearer token is a short-lived MSAL access token — grab from the
#    browser network tab on any /argus/runs request.
git checkout experiment/hitl-base
git submodule update --init --recursive
python -m backend.experiments.harness \
  --arm control \
  --cosmos-account '/subscriptions/.../<acct>' \
  --database hitl_test --collection hitl_eval \
  --bearer-token "$MSAL_TOKEN" \
  --runs 5 --seeds 42,43,44

git checkout experiment/hitl-postreview
git submodule update --init --recursive
python -m backend.experiments.harness --arm A ...   # same args

git checkout experiment/hitl-escalate
git submodule update --init --recursive
python -m backend.experiments.harness --arm B ...   # same args

# 2. Summarise.
python -m backend.experiments.summarize results/
open results/summary.md
```

## Per-arm contract

- **control** — POST /argus/run with the default profile. Oracle labels are
  computed for metrics but no verdicts are sent back to the backend.
- **A** — Same request. After the run, for each finding, POST
  `/argus/findings/{report_id}/{finding_id}/rate {label}` so the planner
  picks the verdict up on the next run via `UserVerdictHistory`.
- **B** — Adds `config_overrides: {finding_evaluator: 'escalation',
  escalation_high_threshold: 0.96, escalation_low_threshold: 0.40,
  escalation_cap: 25}`. After the run, polls `/argus/escalations` for
  this report's pending findings and resolves each via POST
  `/argus/escalations/{report_id}/{finding_id}/resolve {verdict}`. The
  threshold of 0.96 is empirically derived from the smoke-test
  finding that Gemini 2.5 Flash collapses confidence to ~0.95 — tune
  per-model.

## Per-run JSON shape

See the docstring at the top of `metrics.py`. Each `run_<NN>.json` carries
the request, the full report, the per-finding oracle labels, the
escalation queue (for B), and tokens/oracle_seconds.
